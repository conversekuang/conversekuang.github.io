经常看到Python中用urllib进行爬虫的操作，但是终究没有搞清楚有什么urllib，urllib2等模块的关系。最后Python3中的urllib又是和Python2中有什么区别？主要是自己梳理思路。

¶Python2

在python2中获取URL资源有两个模块：urllib 和 urllib2。它们之间是独立的两个模块。而requests 模块是更高级别的模块。

“Content-Type”: “application/x-www-form-urlencoded” // 适用于大部分情况

¶urllib2

如果在HTTP请求中加入header, cookies时，urllib不能满足我们的需求，那么我们可以用urllib2模块实现。

urllib2模块中最常见的就是

  • urllib2.urlopen(url[, data[, timeout[, cafile[, capath[, cadefault[, context]]]]])¶

​ 可以打开一个URL，参数： 字符串类型 或者urllib2.Request类型。

​ 返回：file-like object。该对象处理file-oject对象的方法还要额外的三种：geturl()，getinfo()，getcode()

  • class urllib2.request.Request(url, data=None, headers={}, origin_req_host=None, unverifiable=False, method=None)

    通过加入参数到Request类来构建request ojbect，进而可以带入urllib2.urlopen( )中完成访问。

以上，通过将headers带入到Request类中生成request object，进而通过urllib2.urlopen()完成URL请求。

然而当我们需要在请求中加入proxy、认证、cookie信息等，urlopen()则无法满足该需求。

¶Opener and Handler

通过如下urllib2.urlopen()源码，它其实是一个特殊的opener(也就是模块帮我们创建好的opener)。在urlopen()函数中，首先通过build_opener()返回urllib2.OpenerDirector对象opener，再将该实例赋给global _opener，(该_opener在urlopen()函数外赋值为None)，最后通过**opener.open()**来实现。如果需要处理更特殊的问题，则需要自定义的opener通过open()函数来执行，该方式和执行urlopen()函数有相同的效果。

但是基本的urlopen()不能支持更多的问题处理能力，例如 cookie，proxy等，因此我们要借助Handler和Openner。(可以这么理解，因为模块帮我们定制的是大部分简单问题，遇到特殊问题我们需要具体问题具体分析，因此要使用能处理特定问题的Handler，进而需要生成自定义的opener。)

Handler：处理特定问题的处理器。

Opener：带入Handler通过build_opener()生成OpenerDirector类的一个实例，用于来解决特定问题。

生成Handler后，要生成一个自定义的Opener。需要使用以下方法：

  • urllib2.build_opener([handler, …])

    • 参数：可以是一系列baseHandler或者其子类Handlers，按照给定顺序进行处理。
    • 返回：一个 OpenerDirector 实例。

    因此可以通过添加各种处理问题的handlers，（比如，既要解决代理问题又要解决cookie问题，则将两个handlers一起传入），最后通过build_opener()就可以得到解决该问题的一个 OpenerDirector 实例。

OpenerDirector 对象最常用的方法：

  • OpenerDirector.open(url[, data],[timeout]) 通过传入request object或者URL参数可以进行访问。使用方法类似于urlopen()。如果我们一定要通过urlopen()来使用自定义的opener，那么需要执行urllib2.install_opener()方法。
  • urllib2.install_opener(opener)¶ 将OpenerDirector作为全局Opener。 只有想通过urlopen()函数来操作这个Opener时， 才会执行Install_opener()方法。否则，我们完全可以用OpenerDirector.open()来代替urlopen()。

¶urlparse

urlparse.py主要的作用:

  1. 将URL链接拆分为各部分
  2. 将各部分重新结合为URL链接
  3. 在原有URL链接基础上将相对URL转换成绝对URL链接

常用方法如下：

  • urlparse.urlparse(urlstring[, scheme[, allow_fragments]])¶

将URL解析为6部分scheme://netloc/path;parameters?query#fragment，可以通过index来获取各部分值。

Attribute Index Value Value if not present
scheme 0 URL scheme specifier scheme parameter
netloc 1 Network location part empty string
path 2 Hierarchical path empty string
params 3 Parameters for last path element empty string
query 4 Query component empty string
fragment 5 Fragment identifier empty string
username User name None
password Password None
hostname Host name (lower case) None
port Port number as integer, if present None
  • urlparse.urlsplit(urlstring[, scheme[, allow_fragments]])¶

将URL按模式进行拆分，与urlparse()相比较，只是没有把参数params从URL中分割出来。如图所示：

  • urlparse() 与 urlsplit() 返回结果的处理

urlparse() 与 urlsplit() 方法返回的结果都是 tuple 类型。该返回值可以通过geturl()方法得到重新组合后的URL，该URL与原URL稍有不同，sheme属性统一为小写，此外，空参数，请求，和锚点都将被移除。

  • urlparse.urljoin(base, url[, allow_fragments])¶

    通过base链接将相对链接（url参数）转换成绝对链接（返回值）。相对链接一般是从base链接中获取的协议模式、网络地址、以及（部分）路径信息，最后获得绝对URL链接。

    函数urljoin在通过为URL基地址附加新的文件名的方式来处理同一位置处的若干文件的时候格外有用。需要注意的是，如果基地址并非以字符/结尾的话，那么URL基地址最右边部分就会被这个相对路径所替换。如果希望在该路径中保留末端目录，应确保URL基地址以字符/结尾。

以上urllib，urllib2以及urlparse，这三个独立的模块经常在Python2中配合使用。urlparse注重的是URL链接拆分与组合；urllib可以实现简单的资源访问；urllib2更加丰富以及复杂的请求。urllib2与urllib不是替代或者升级关系，通常在Python2中他们经常需要配合使用完成URL资源的请求。

¶Python3

在Python2中，资源请求的模块比较分散，因此在Python3中进行整合。Python3中，urllib包中只有4个模块。

urllib主要分为以下四个模块

  • 第一个模块 request，它是最基本的 HTTP 请求模块，我们可以用它来模拟发送一请求，就像在浏览器里输入网址然后敲击回车一样，只需要给库方法传入 URL 还有额外的参数，就可以模拟实现这个过程了。
  • 第二个 error 模块即异常处理模块，如果出现请求错误，我们可以捕获这些异常，然后进行重试或其他操作保证程序不会意外终止。
  • 第三个 parse 模块是一个工具模块，提供了许多 URL 处理方法，比如拆分、解析、合并等等的方法。
  • 第四个模块是 robotparser，主要是用来识别网站的 robots.txt 文件，然后判断哪些网站可以爬，哪些网站不可以爬的，其实用的比较少。

Python3在用法上和Python2中的函数用法基本相同。具体介绍参考崔庆才–Python3网络爬虫发开实战的这部分Python3的Urllib，不再赘述。
