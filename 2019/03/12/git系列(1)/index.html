Git学习

config设置

git使用的最小配置需要设置user.name 与 user.email

  • 配置

git config [–local | --global | --system] user.name 'Your name’
git config [–local | --global | --system] user.email ‘Your email’

字段与值之间没有“=”号

  • 查看

git config --list [–local | --global | --system]

git config -l

  • 作用域

    local： 只对本仓库有效
    global: 当前登录用户的所有仓库
    system: 对系统的所有用户

  • 优先级

    local > global > system

完整的git提交过程

  • git常规操作流

git init 初始化仓库

git add file 添加到暂存区，并跟踪文件

git status 查看暂存区状态

git commit -m “comments” 将暂存区文件提交到仓库

git log 查看提交历史信息

或者

git commit -am “comments” 直接将工作区文件提交到仓库（命令对新创建的文件无效，也就是新创建的文件不能使用这个命令提交到版本历史中，只有对跟踪文件的才能用 -am）

  • 三个区域

工作目录： 在当前的工作路径上进行操作文件

暂存区： 将目前的工作状态进行暂时的保存，在提交与修改之间留下缓冲去，可试用多种方案的比较。

仓库： 将暂存区的工作状态提交本地仓库保存。

  • 根据资料

git add -u 将已经跟踪的文件更新到暂存区**(修改、删除**的文件)

git add . 将文件更新到暂存区(新增、修改 的文件)

git add -A 将所有文件更新到暂存区**(新增、修改、删除**的文件)

实践结果：经测试在windows 2版本下，git add . 与 git add -A 具有相同的功能。

工作区文件 修改名称（重命名）

以index 文件修改为 index.html

  • 按照git完整流程修改名称，分为三步

    mv index index.html 在工作路径上修改名称

    git rm index 删除修改前原来old文件

    git add index.html 新加修改后的new文件

    以上的三个操作，git可以智能地发现是修改文件名的步骤。因此，在暂存区显示的是renamed提示

  • 可以变成两步：

    mv index index.html 在工作路径上修改名称

    git add index.html 新加修改后的new文件

    其实不用执行 git rm 指令来删除文件，因为git足够智能，能够判定我们在修改名称

  • 一个步骤可以代替以上三个步骤：

    git mv oldname newname

    git status 同样会显示 renamed: index -> index.html

【大小写敏感问题】:

  • 使用Windows系统，系统的大小写是不敏感的。
    git 配置文件中默认core.ignorecase 配置是 true
    • 基于此配置做 git mv readme Readme 操作的话，可以在 git status 看到文件名的变化。
    • 但是，同样基于此配置，做 mv readme Readme 操作后，再使用 git status 查看文件状态，发现 status 仍为 clean 的状态。
    • 当 core.ignorecase 配置修改为 false 时，两个操作的执行结果正好相反。
  • 当切换到Ubuntu这种大小写敏感的系统上，不管 core.ignorecase 配置是什么，使用 git mv 和 mv 命令的结果都是一样的。 (from 评论区 “万想")

查看commit 提交的历史信息

git log: commit提交历史包含较多无需不关注的信息

git log -n4 --oneline --graph git所在分支的最近4次提交记录，简洁的查看版本历史、以图的方式显示

git log -n4 --oneline --graph --all 本仓库所有分支的最近4次提交记录，简洁的查看版本历史、以图的方式显示

gitk --all 图形化界面进行查看

git branch -av 查看仓库的所有分支（远程分支、本地分支）

git branch -v 查看本地分支（只有本地分支）

git checkout -b temp commit_id 基于commit_id历史提交版本，新建temp分支并切换到该分支，

【命令带参数用–或-】:

git 命令带参数，什么时候用 – ？什么时候用 - 呢？

发现似乎是单字母的参数是 ‘-’，非单字母的参数是’–’

gitk图形化界面查看提交历史信息

【重点说明几个】

Author： 作者

Committer： 提交人

Author 和Committer是不同人的情况：当基于A中某分支的某个commit -> B建立新的分支，并在此new分支上进行操作修改最后进行提交。这时，Author是 A， Committer 是 B。

patch ： 显示的修改的信息

tree： 显示的是目录信息以及文件信息(完整的文件信息)
