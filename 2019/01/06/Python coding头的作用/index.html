Python coding头的作用

Python2中乱码是让人非常头疼的问题，怎么样杜绝出现乱码错误？这个牵扯很多方面的知识，Unicode方面，编码方式等。后面的会陆续更新。本篇文章主要讲的是Python coding头的作用。

¶基础知识

Python2 中字符串有两类：unicode 和 str 类型

unicode类型与str类型之间的转换

unicode类型通过encode()可以转换为str类型

str类型通过decode()可以转换为unicode类型

为什么说unicode是str类型各个编码方式的中间状态？

因为unicode字符串内容代表unicode的位置信息，该位置信息在unicode字符集中是不变的。那么当然是“以不变应万变”，通过unicode字符串将字符转换成不同编码。比如，我们有一个str类型的字符串对象 A，它是utf8编码的。如何将它转换成gbk编码的字符串对象 C呢？

首先通过B = A.decode(‘utf8’)操作把A变为unicode字符串，再通过C = B.encode(‘gbk’)操作把B变为gbk编码的str字符串。也就是：C = A.decode('utf8').encode('gbk')。我们将utf-8编码的字符串A转换成gbk编码的字符串C。可以看到，B对象就是unicode字符串类型，在编码转换中起到了桥梁的作用。

我们经常使用的集成开发软件，编辑器基本上都设置了默认的编码的编码方式。所以在我们在写完代码保存时，这些工具按照默认的编码方式来保存文件。 比如sublime默认的编码方式为UTF-8，当我们在windows的记事本中写完保存时，默认的是ANSI编码方式（也就是gbk编码）。可以试一下，当我们在文本中输入中文但是选择字符集不包含汉字的编码方式，会出现什么情况？

可以看出，当选择字符集中不包含汉字的编码方式，无法保存文件。因为对于字符集中不存在的字符，无法进行编码来转换成二进制文件保存，因此会出现错误。这里，我们要说的就是文件保存时已经带有默认的编码方式，而当采用与文件编码方式不同的解码方式读取时，就会产生冲突，要么乱码或者就直接报错无法找到。

script.py脚本采用sublime中的utf-8默认编码方式，而python的解释器默认采用ASCII编码进行解释（根据 PEP 263 文档），显然解码方式与文件编码方式冲突，就会产生错误。主要是因为ASCII并不包括汉字，因此无法解释汉字对应的十六进制数。

¶脚本的执行

#!/usr/bin/python 或 #!/usr/bin/env python

  • 注释在windows中没有作用。

    我们在命令行运行脚本仍然采用 python xxx.py。系统能够识别python的解释器是因为在安装时候设置了环境变量path值，该值为python解释器路径。

  • 注释在Linux或者Unix下有效

    在Linux或者Unix下，相当于告诉脚本到指定用什么解释器运行脚本以及解释器所在的位置。有以下两种方式：

    #!/usr/bin/python

    是告诉操作系统执行这个脚本的时候，调用 /usr/bin下的python解释器；

    #!/usr/bin/env python

    这种用法是为了操作系统防止用户没有将python 装在默认的 /usr/bin 路径里当系统看到这一行的时候，首先会到env设置里查找python的安装路径，再调用对应路径下的解释器程序完成操作。

¶脚本的编码

Python解释器将根据脚本中设定的编码信息来翻译文件，Python解释器将ASCII作为默认的编码方式。

如果我们需要使用编码头，那么必须将这个coding头写在文件的第一行或者第二行。由于第一行指出解释器位置，那么第二行则是说明编码方式。所以常见到这样的写法:

此外，根据官方文档 PEP 263 提到了兼容windows中的特别情况。下面进行说明。

在windows记事本中，编码方式有4种：

  • ANSI：在简体中文Windows操作系统中，ANSI 编码代表 GBK 编码。在新建文件中默认编码是ANSI。

  • unicode：采用小端模式utf-16进行编码。在文件开头加上**\xff\xfe**

  • unicode big endian：采用大端模式utf-16进行编码。在文件开头加上**\xfe\xff**

  • UTF-8：这个UTF-8是带BOM的，也就是我们在选择这个编码方式时候，在标准UTF-8编码的基础上，文件还会在开头处加上一个串16进制数：\xef\xbb\xbf

通过以上例子，windows的记事本除了对文件内容编码外还会加上编码头。

值得注意的是，windows记事本中的UTF-8不是标准的UTF-8。因为它会在文件开头加上**’\xef\xbb\xbf’**，通常称为BOM。

为了照顾特殊的Windows中的带BOM(’\xef\xbb\xbf’)的UTF-8:

  1. 如果python文件是带BOM的UTF-8编码文件，即使文件开头没有编码方式的说明，也可以自动默认是UTF-8编码。
  2. 如果python文件是带BOM的UTF-8编码文件，但是文件头中的编码不是UTF-8，这会造成，声明的编码和实际编码不一致，所以自然会报错。

¶sys.setdefaultencoding()函数

sys.getdefaultencoding() 与 sys.setdefaultencoding()

sys.setdefaultencoding() 用于设定自动转换字符串的编解码方式。默认的编解码方式是ASCII。

字符串转换的方式是进行编解码，那么有两种：显式与隐式。

  • (1) 显式的进行编解码

windows命令行中，默认是采用gbk编码方式。那么如何转换成utf-8编码方式？

可以看出，在转换时，我们需要将str类型的字符串先解码再编码。s.decode('gbk').encode('utf-8)

  • (2) 隐式的进行编解码

而在隐式的编解码，我们就需要sys.setdefaultencoding()来设置默认的编码方式。转化时，只需要通过encode()函数来实现目标编码的字符串。因为，python解释器自动的对字符串按照编解码方式进行decode()，转换成unicode类型的字符串。

总结：

coding 头： 是python解释器如何对二进制脚本文件读取方式的定义，默认采用ASCII解释脚本文件。coding头是针对python解释器读取文件的设置。

sys.setdefaultencoding()：设置字符串默认编解码的方式，默认采用ASCII。可以根据需要设置为特定的编码方式。该函数是python解释器针对字符串编码的设置。
